# 在线实验与涨指标

## 第 15 章 在线实验与涨指标

离线实验回答"模型在固定数据上是否更好"，线上实验回答"真实用户与整条系统交互后是否更好"。两者之间隔着候选变化、规则、延迟、反馈闭环和产品行为，不能用一个离线 AUC 直接跨过去。

### 15.1 先写实验问题

推荐的指标树从转化链路开始：`曝光 -> 点击 -> 阅读/播放 -> 点赞、收藏、转发、评论`。点击率以曝光为分母，点赞/收藏/转发率通常以点击为分母。再往上是北极星：DAU/MAU、人均消费时长或内容数，以及作者侧的发布渗透率和人均发布量。离线实验只能筛选候选方案，发布仍要经过小流量 A/B 和全流量上线。

一个可执行的实验设计至少写清：

```text
假设：新增 GNN 召回能补到旧通道缺失的长尾正例
随机化单元：user_id
主指标：有效消费时长 / user
过程指标：新增通道独占召回、CTR、有效播放
护栏：P99、投诉率、作者集中度
最小可检测效应：+0.3%
观察窗口：14 天，并覆盖完整周周期
```

主指标只能有少量几个。先看几十个指标，再挑上涨的写结论，会产生多重检验和汇报偏差。

### 15.2 稳定随机分桶

最常见的用户级分桶：

```text
bucket = hash(user_id, experiment_id, salt) mod B
```

要求：

- 同一用户跨请求稳定落桶；
- 桶大小近似均匀；
- 不同实验使用不同 salt；
- 服务端、日志和离线分析采用同一实现；
- 分桶版本可追溯。

Python 内置 `hash()` 会受进程随机种子影响，不适合跨机器实验分桶。应使用 SHA-256、MurmurHash 等固定算法。

随机化单元不一定是用户。社交网络实验可能需要 cluster randomization，商家策略可能按 seller，创作者策略可能按 author。随机化单元应对应干预发生和互相影响的边界。

### 15.3 分层：同层互斥，不同层正交

多个实验共享流量时，常把召回、粗排、精排、重排、界面、广告划成独立层。

- 同层实验互斥：同一用户不能同时使用两个互相替代的精排模型；
- 不同层实验正交：召回实验和界面实验可独立分桶，用户交集是预期现象。

正交的前提是交互效应可以接受。新的召回只产生旧精排没见过的候选时，召回与精排可能强交互，不能机械地假设 `1 + 1 = 2`。重要组合应增加 factorial cell 或专门做交互检验。

### 15.4 上线前先做 A/A

A/A 不改变策略，只检查实验基础设施。至少看：

- Sample Ratio Mismatch：实际分流比例是否符合配置；
- 实验组和对照组的实验前指标是否平衡；
- 曝光、日志和指标归因是否漏数；
- 用户是否跨桶；
- 指标方差是否符合历史水平；
- 机器人、内部用户和异常流量是否一致过滤。

SRM 可用卡方检验：

```math
\chi^2
=\sum_b \frac{(O_b-E_b)^2}{E_b}.
```

SRM 显著时先修分流或日志，不解释业务指标。随机化已经坏了，后面的 `p-value` 没有可信基础。

### 15.5 效应、区间和样本量

两组均值差：

```math
\Delta=\bar X_T-\bar X_C.
```

结果至少报告：

- 绝对差与相对差；
- 置信区间；
- 样本量和实验时长；
- 预先定义的主指标；
- 护栏是否越界。

`p < 0.05` 不等于收益有业务价值。反过来，不显著也不等于没有收益；如果区间仍覆盖 `+0.5%` 这类重要效果，只能说数据还不足。

实验前要用基线均值、方差、显著性水平、power 和 MDE 估算样本量。CUPED 可利用实验前同类指标降方差：

```math
Y_{\text{cuped}}
=Y-\theta(X-\mathbb E[X]),
\qquad
\theta=\frac{\operatorname{Cov}(Y,X)}
{\operatorname{Var}(X)}.
```

`X` 必须在实验前产生。拿实验中的特征做协变量会引入处理后的偏差。

### 15.6 Holdout、推全和反转实验

实验层回答单个策略的增量，长期 Holdout 回答一段时间内多次发布的累计价值。Holdout 用户持续使用旧策略，不参与这条业务线的新实验。

推全后还可保留小流量反转桶：

```text
大部分用户：新策略
反转桶：旧策略
```

点击等指标很快变化，留存和生态指标有滞后。反转桶让系统腾出普通实验桶，同时继续观察长期差异。

Holdout 也会老化。长期不更新的旧策略可能与当前系统不兼容，因此需要明确重置周期、版本边界和安全下限。

### 15.7 冷启动实验为什么难

新品策略同时影响用户和作者：

- 用户侧：新品 CTR、交互、消费时长和负反馈；
- 作者侧：发布渗透率、人均发布量、首个有效反馈等待时间。

只按用户分桶时，同一批新品会在实验组和对照组之间争抢流量。只按作者分桶时，不同作者的新品还会与全部老内容竞争；实验中的竞争格局与推全后的格局不同。

设计时必须逐项回答：

1. 两组新品是否会互相抢固定探索预算？
2. 新品与老内容怎样竞争？
3. 同时隔离用户和作者是否把可用内容池砍得太小？
4. 保量策略是否改变了其他候选的曝光？
5. 用户侧收益和作者侧收益由哪个随机化单元估计？

这类网络干扰下，不存在无代价的完美分桶。报告中应明确 estimand：估计的是当前局部流量下的直接效应，还是接近推全后的总体效应。

### 15.8 怎样系统地找增长点

先沿漏斗定位，再选模型。

召回可检查：

- 正例是否根本没进入候选；
- 双塔是否缺少自监督或多行为信号；
- 长尾兴趣是否被 Last-N 截断；
- 新通道是否只重复旧通道；
- 曝光过滤是否误杀。

排序可检查：

- 特征覆盖率和线上缺失；
- 粗排对精排 top item 的误杀；
- 样本偏差与标签成熟；
- 多目标融合与校准；
- 长序列、实时特征和增量训练。

列表可检查：

- 重复和类目集中；
- 探索预算；
- 特殊人群是否被总体模型忽略；
- 广告、运营和安全规则是否抵消模型收益。

### 15.9 特殊人群

新用户、低活用户和少数人群的行为分布可能与主流用户不同。可采用三层干预：

- 内容池：按人群的质量与留存贡献构造弱个性化池；
- 策略：减少广告和高风险探索，调整融分权重；
- 模型：共享大模型加残差小模型、多 expert gating，或单独校准。

不建议为每个人群长期维护一套完整大模型。主模型持续升级后，没人维护的分群模型很快落后。共享底座加轻量适配更容易保持一致。

### 15.10 利用关注、转发和评论

这些交互还会改变关系、创作和站外回流，不能只塞进一列排序标签。

关注有两侧价值：

- 用户关注更多合适作者后，平台关系更稳定；
- 新作者获得首批粉丝后，更可能继续发布。

可以构造促关注内容池、U2A2I 显式/隐式关注召回，并让促关注权重随用户已关注数或作者已有粉丝数递减。

融分项可以写成 `w(f_u) p_follow(u,i)`：用户已关注作者越少，`w(f_u)` 越大。作者侧也可用随粉丝数递减的权重，帮助低粉作者获得首批关注。

转发的价值取决于是否带来站外回流。单纯提高转发率可能伤害点击，应识别能带来回流的用户和内容，再做条件化提权。

课程中的 KOL 信号看的是该用户历史转发实际带回多少站外流量，不是他在本站的粉丝数。识别后只对这类用户增加促转发融分项或召回通道。

评论能提高作者反馈和社区讨论，但要区分数量与质量。促评论策略还要观察争吵、举报和低质互动，不能只把评论数当正收益。

新内容已有评论越少，促评论权重可以越高；对经常产出高赞评论的用户，也可提供更适合讨论的内容池。两类策略分别服务作者反馈和社区质量。

### 15.11 本章自测

1. 为什么不同实验需要不同 salt？
2. 同层互斥和不同层正交分别解决什么问题？
3. SRM 显著时为什么不能继续解释业务指标？
4. 不显著是否等于策略无效？
5. CUPED 的协变量为什么必须来自实验前？
6. 冷启动实验为什么容易出现组间干扰？
7. 为什么不建议为每个特殊人群维护完整大模型？
8. 关注、转发和评论为什么不能只当普通正标签？
