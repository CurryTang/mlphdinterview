# 列表决策：重排、探索与冷启动

## 第 10 章 重排、多样性与规则

### 10.1 列表多了一层 item-item 约束

精排逐项预测收益。若前十个候选都是同一作者讲同一件事，它们每个都可能分高，列表体验却很差。

重排的输入是候选集合和精排分数，输出是一个受约束的有序列表。它需要同时考虑：

- 单项收益；
- 候选之间的相似性；
- 类目、作者、内容形态覆盖；
- 广告、运营和安全规则；
- 前后位置的联动；
- 用户已经看过的内容。

### 10.2 多样性怎样量化

最简单的类目覆盖：

```math
\operatorname{Coverage@K}
=|\{category(i): i\in topK\}|.
```

也可以计算列表内平均相似度：

```math
\operatorname{ILS}
=\frac{2}{K(K-1)}
\sum_{i<j}\operatorname{sim}(i,j).
```

ILS 越低通常越多样，但"不相似"不一定是好事。随机塞入不相关内容能让 ILS 很低，却会毁掉体验。多样性必须与相关性或预估收益一起看。

相似度也不只一种。类目相同、作者相同、文本相似、embedding 接近代表不同重复感。业务上往往需要组合。

### 10.3 MMR

MMR 每次从未选集合中挑一个候选：

```math
i^*
=\arg\max_{i\in R}
\left[
\theta\,r_i
-(1-\theta)\max_{j\in S}\operatorname{sim}(i,j)
\right].
```

`r_i` 是精排收益，`S` 是已选集合，`θ` 控制收益与多样性的权衡。

MMR 直观、易调，也容易加入硬过滤。它的局限是贪心，且只看候选与最相似已选项，不能完整表达集合结构。

### Quick Coding：MMR 重排

给定候选相关性和两两相似度，贪心选出 top-k。要支持只提供一个方向的相似度，并在分数相同时给出确定性顺序。

题目：[[BusinessAlgorithm09 Quick Coding.md#QC07 MMR 重排|QC07 MMR 重排]]。

### 10.4 DPP

DPP 用一个正半定核矩阵 `L` 描述质量和相似性：

```math
P(S)\propto\det(L_S).
```

若两个候选非常相似，对应向量接近线性相关，行列式变小；候选质量高且方向多样时，行列式较大。

常把：

```math
L_{ij}=q_i\,S_{ij}\,q_j
```

其中 `q_i` 是质量，`S_{ij}` 是相似度。求精确最优子集昂贵，线上通常用贪心和增量矩阵更新。

DPP 的数学形式漂亮，但服务里还要面对数值稳定、候选规模和硬规则。很多团队最终使用的是 DPP 思想加业务近似，而非教科书里的完整采样。

### 10.5 滑动窗口

长列表不一定要求全局去重。用户对相邻内容的重复最敏感，可以只惩罚最近 `W` 个已选候选：

```text
选第 t 个候选时，只与位置 [t-W, t-1] 比相似度
```

这样计算更便宜，也允许同一主题隔一段时间重新出现。`W` 应根据屏幕形态和消费节奏调，而不是固定照搬。

### 10.6 规则怎样进入重排

规则可分三类。

硬约束必须满足，例如安全、库存、法律、广告上限。先过滤或在构造列表时严格检查。

软约束允许权衡，例如作者打散、类目覆盖、长尾扶持。可作为 penalty 或 reward。

配额约束要求某类内容在区间内出现若干个，例如冷启动内容至少 1 个、广告不超过 2 个。可用分桶选取、整数规划近似或带约束贪心。

把所有规则都塞进一条长 if-else 很快失控。更稳的做法是把规则声明成可配置约束，记录每个候选被哪条规则删掉，支持离线回放。

### 10.7 搜索与推荐的多样性不一样

搜索 query 通常有明确答案。用户搜"iPhone 15 参数"，前几条必须相关，不能为了多样性塞入安卓手机。

推荐可以扩展兴趣，但也不能无限探索。两者都需要多样性，约束优先级不同：

```text
搜索：相关性底线 > 质量/时效 > 多样性
推荐：预估兴趣与长期体验共同约束多样性
```

## 第 11 章 冷启动、探索与长期反馈

### 11.1 Embedding 只给初始估计

新物品没有交互，协同召回找不到它；新用户没有历史，个性化模型也无从判断。即使能用内容 encoder 生成 embedding，系统仍不知道真实消费反馈。

冷启动需要解决两个问题：

1. 用现有信息给出初始估计；
2. 在可控风险下获得真实反馈。

只做第一个，没有探索，模型永远难以纠正初始误判。

### 11.2 新物品候选

常见通道：

- 类目、标签、作者与地域；
- 文本/图像/多模态相似；
- 聚类后投放到匹配兴趣簇；
- Look-Alike 人群扩散；
- 运营或质量审核后的新品池。

Look-Alike 从少量种子用户出发，寻找画像或行为相近的人群。它依赖种子质量，也容易把早期偏差放大。

### 11.3 新用户

新用户可利用：

- 地区、语言、设备和入口；
- 首屏交互；
- 显式兴趣选择；
- 上下文热门；
- 相似匿名 session。

新用户最宝贵的是前几次行为。系统应快速更新短期兴趣，不要让默认热门持续太久。

### 11.4 流量从哪里注入

冷启动候选可以在不同层进入：

- 召回注入：风险小，但可能被粗排和精排全部淘汰；
- 粗排后保量：更容易获得曝光，质量控制要求更高；
- 精排后插入：曝光稳定，可能明显伤害主列表；
- 重排配额：最可控，也最需要明确预算和护栏。

注入越靠后，曝光越有保证，主列表受损的风险也越大。高质量审核池可以在重排保量；来源不明的新内容更适合从召回层小流量试探。

### 11.5 探索与利用

利用选择当前估计最好的内容，探索选择不确定但可能有潜力的内容。

一个简单 UCB 形式：

```math
\operatorname{UCB}_i
=\hat\mu_i
+c\sqrt{\frac{\log t}{n_i}}.
```

`μ_i` 是当前收益估计，`n_i` 是曝光次数。曝光少的物品获得不确定性 bonus。

推荐里的收益依赖用户上下文，物品会过期，反馈还有延迟和位置效应，所以 UCB 公式只能说明探索方向。即使不用 bandit，也要给高不确定性候选一小段受控流量，否则系统无法收集纠正初始估计的数据。

### 11.6 冷启动评价

总体 CTR 会被成熟物品淹没。至少分开看：

- 新品进入召回的比例；
- 首次曝光等待时间；
- 达到最小反馈量所需流量；
- 新品分层 CTR/时长/负反馈；
- 探索对主列表的损伤；
- 经过探索后留存下来的优质新品比例。

还要按作者、类目和质量先验分层。把所有新品混在一起平均，会让策略看起来毫无规律。

### 11.7 反馈闭环

推荐系统训练在自己的曝光上。模型偏好热门，热门获得更多曝光，又变得更热门。长期结果可能是内容越来越窄，新作者越来越难进入。

缓解方法包括：

- 随机或不确定性探索；
- 热门度去偏和 propensity weighting；
- 新品保量；
- 长期 Holdout；
- 创作者侧生态指标；
- 训练样本中的曝光校正。

这些方法会牺牲一部分短期点击。是否值得，是产品和生态决策，不是纯模型问题。
