# 冷启动、探索与长期反馈

## 第 14 章 冷启动、探索与长期反馈

### 14.1 Embedding 只给初始估计

新物品没有交互，协同召回找不到它；新用户没有历史，个性化模型也无从判断。即使能用内容 encoder 生成 embedding，系统仍不知道真实消费反馈。

冷启动需要解决两个问题：

1. 用现有信息给出初始估计；
2. 在可控风险下获得真实反馈。

只做第一个，没有探索，模型永远难以纠正初始误判。

### 14.2 新物品候选

常见通道：

- 类目、标签、作者与地域；
- 文本/图像/多模态相似；
- 聚类后投放到匹配兴趣簇；
- Look-Alike 人群扩散；
- 运营或质量审核后的新品池。

Look-Alike 从少量种子用户出发，寻找画像或行为相近的人群。它依赖种子质量，也容易把早期偏差放大。

简单通道先解决"能被找到"：

- 同作者：把作者的新内容召回给近期看过该作者的用户；
- 同类目/标签：根据用户历史类目偏好做匹配；
- 内容向量：用标题、图像和视频表示做近邻；
- 新品池：通过质量审核后按场景或人群取样。

双塔也要专门处理新 ID。物品还没有自己的 ID embedding 时，可以先用共享 default embedding，或取若干内容相似、高曝光物品的 ID embedding 均值。再按 1 小时、6 小时、24 小时等年龄建多个 ANN 池；这些池共享模型，只增加索引和检索成本。

聚类召回先训练多模态内容相似度模型，再对内容向量做 k-means。训练正 pair 可选同二级类目且 ItemCF 相似度高的高曝光物品，负例从文本信息足、质量合格的其他物品中采；这样模型学的是内容相似，不是图文质量。线上从用户 Last-N 种子 item 找最近 cluster，再从 `cluster -> 按时间倒排的新品` 取候选。簇太粗会混意图，太细又会变成稀疏 ID。

Look-Alike 的种子可以是已经对新品产生高质量交互的用户。一个直接实现是近线维护这些种子用户 embedding 的均值，把它当成新品人群向量；在线用当前用户向量在新品向量库里做 ANN。每次有新交互就更新均值。种子少时画像规则或小型分类器更稳，还要限制扩散规模，避免第一批偶然用户定义全部受众。

### 14.3 新用户

新用户可利用：

- 地区、语言、设备和入口；
- 首屏交互；
- 显式兴趣选择；
- 上下文热门；
- 相似匿名 session。

新用户最宝贵的是前几次行为。系统应快速更新短期兴趣，不要让默认热门持续太久。

### 14.4 流量从哪里注入

冷启动候选可以在不同层进入：

- 召回注入：风险小，但可能被粗排和精排全部淘汰；
- 粗排后保量：更容易获得曝光，质量控制要求更高；
- 精排后插入：曝光稳定，可能明显伤害主列表；
- 重排配额：最可控，也最需要明确预算和护栏。

注入越靠后，曝光越有保证，主列表受损的风险也越大。高质量审核池可以在重排保量；来源不明的新内容更适合从召回层小流量试探。

流量调控通常经历 boost、动态保量和差异化保量。Boost 实现简单，但曝光量对系数很敏感。动态保量根据 `已发布时间 / 目标时间` 和 `已有曝光 / 目标曝光` 调权，越接近截止时间且缺口越大，提权越强。差异化保量再根据内容质量和作者历史质量设置不同目标，避免所有新品不分质量拿同样流量。

### 14.5 探索与利用

利用选择当前估计最好的内容，探索选择不确定但可能有潜力的内容。

一个简单 UCB 形式：

```math
\operatorname{UCB}_i
=\hat\mu_i
+c\sqrt{\frac{\log t}{n_i}}.
```

`μ_i` 是当前收益估计，`n_i` 是曝光次数。曝光少的物品获得不确定性 bonus。

推荐里的收益依赖用户上下文，物品会过期，反馈还有延迟和位置效应，所以 UCB 公式只能说明探索方向。即使不用 bandit，也要给高不确定性候选一小段受控流量，否则系统无法收集纠正初始估计的数据。

### 14.6 冷启动评价

总体 CTR 会被成熟物品淹没。至少分开看：

- 新品进入召回的比例；
- 首次曝光等待时间；
- 达到最小反馈量所需流量；
- 新品分层 CTR/时长/负反馈；
- 探索对主列表的损伤；
- 经过探索后留存下来的优质新品比例。

还要按作者、类目和质量先验分层。把所有新品混在一起平均，会让策略看起来毫无规律。

课程把指标分成三侧：作者侧看发布渗透率（当日发布人数 / DAU）和人均发布量（当日发布内容数 / DAU）；用户侧看新品分层 CTR/交互率以及大盘时长、DAU、MAU；内容侧看规定窗口内达到高热阈值的新品占比。扶持策略可能让作者指标上涨、用户消费下降，三侧必须一起报。

### 14.7 反馈闭环

推荐系统训练在自己的曝光上。模型偏好热门，热门获得更多曝光，又变得更热门。长期结果可能是内容越来越窄，新作者越来越难进入。

缓解方法包括：

- 随机或不确定性探索；
- 热门度去偏和 propensity weighting；
- 新品保量；
- 长期 Holdout；
- 创作者侧生态指标；
- 训练样本中的曝光校正。

这些方法会牺牲一部分短期点击。是否值得，是产品和生态决策，不是纯模型问题。

### 14.8 本章自测

1. 有内容 embedding 后，冷启动为什么仍然存在？
2. 冷启动候选在召回后和精排后注入，各有什么风险？
3. UCB 的不确定性 bonus 为什么会随曝光增加而下降？
4. 哪些指标能反映新品真正获得了机会？
5. 如何判断系统正在形成热门反馈闭环？
6. 聚类召回和 Look-Alike 分别从哪一侧解决新品冷启动？

<details>
<summary>参考答案</summary>

1. 内容向量只能提供初始相关性，仍缺少点击率、满意度、人群差异和质量反馈；系统还需要给新品获得可观测反馈的流量。
2. 召回后注入覆盖更广，但会消耗大量粗排算力并混入低质候选；精排后注入成本低，却可能绕过模型质量控制并伤害列表。
3. 曝光越多，均值估计的方差越小，置信区间收窄，因此探索 bonus 应逐渐下降，把流量转给真正表现好的 item。
4. 看获得最低曝光门槛的新品比例、首次有效反馈时间、不同类目覆盖和曝光后的质量，而不只看新品总曝光量。
5. 观察曝光集中度、热门 item 的入选概率是否随历史曝光持续上升，以及控制质量后 popularity 特征是否仍主导排序；随机或探索桶可作为对照。
6. 聚类召回从 item 内容侧给新品分配兴趣簇；Look-Alike 从少量已反馈用户出发扩展相似受众。前者先解决可召回性，后者利用早期人群信号扩大流量。
